# 跨平台打包工具设计方案 (PyApp CLI)

## 一、概述

### 1.1 背景

当前项目已验证 Android 打包方案可行性（基于 Chaquopy + FastAPI + WebView），同时规划了 Windows/Linux 的打包方案。但现有构建流程分散在多个脚本中，缺乏统一的开发体验。

### 1.2 目标

打造类似 Briefcase 的跨平台开发体验：

```
一份 Python 代码 → 一行命令 → 三平台安装包
```

**核心目标**：

- 统一 CLI 命令接口
- 统一配置文件管理
- 统一开发调试流程
- 自动化环境配置

### 1.3 与 Briefcase 对比

| 维度         | Briefcase        | PyApp CLI (本方案)        |
| ---------- | ---------------- | ---------------------- |
| UI 框架      | Toga (原生组件)      | WebView + FastAPI      |
| 适用场景       | 桌面应用为主           | 服务型应用为主                |
| Android 支持 | Chaquopy         | Chaquopy               |
| Windows 支持 | MSI 安装包          | ZIP 解压即用               |
| Linux 支持   | deb/rpm/AppImage | tar.gz + systemd       |
| 热更新        | 不支持              | 支持 (VERSION 文件机制)      |
| 开发模式       | `briefcase dev`  | `pyapp dev` (文件监听+热重载) |

## 二、命令设计

### 2.1 命令总览

```powershell
pyapp <command> [platform] [options]
```

| 命令        | 说明        | 示例                                   |
| --------- | --------- | ------------------------------------ |
| `init`    | 初始化新项目    | `pyapp init myapp`                   |
| `create`  | 创建平台项目结构  | `pyapp create android`               |
| `build`   | 构建安装包     | `pyapp build android`                |
| `run`     | 安装并运行应用   | `pyapp run android`                  |
| `dev`     | 开发模式（热重载） | `pyapp dev android`                  |
| `package` | 打包发布版     | `pyapp package android`              |
| `deploy`  | 部署到目标设备   | `pyapp deploy android 192.168.1.100` |
| `setup`   | 安装平台依赖环境  | `pyapp setup android`                |
| `logs`    | 查看或管理日志   | `pyapp logs --lines 50`              |

### 2.2 平台参数

| 参数        | 说明         |
| --------- | ---------- |
| `android` | Android 平台 |
| `windows` | Windows 平台 |
| `linux`   | Linux 平台   |
| `all`     | 所有平台       |

### 2.3 命令详解

#### 2.3.1 `pyapp init`

初始化新项目，生成标准项目结构。

```powershell
pyapp init myapp [--template basic|fastapi]
```

**产出**：

```
myapp/
├── pyproject.toml          # 项目配置（版本、依赖、平台配置）
├── src/                    # Python 源码目录
│   └── myapp/              # Python 包
│       ├── __init__.py
│       ├── __main__.py     # 统一入口
│       ├── app.py          # FastAPI 应用
│       └── resources/      # 静态资源目录
├── frontend/               # 前端项目（可选，独立管理）
│   ├── package.json        # npm 依赖
│   ├── src/                # 前端源码
│   │   ├── App.vue
│   │   └── main.ts
│   └── dist/               # 编译产物（gitignore）
│       ├── index.html
│       └── assets/
│           ├── index.js
│           └── index.css
└── bundles/                # 平台打包产物（自动生成）
    ├── android/
    ├── windows/
    └── linux/
```

> **说明**：
>
> - 版本信息和依赖全部集中在 `pyproject.toml` 中
> - 使用 `src/` 目录布局，符合现代 Python 项目规范（PEP 517/518）
> - `resources/config/` 存放配置文件等需要入库的资源
> - `frontend/` 独立管理前端项目，`dist/` 编译产物不入库
> - `bundles/` 目录存放各平台打包产物，构建时自动同步前端资源
> - 与 Briefcase 目录结构对比：
>   | Briefcase              | PyApp                  | 说明     |
>   | ---------------------- | ---------------------- | ------ |
>   | `src/`                 | `src/`                 | 源码目录   |
>   | `src/{app}/resources/` | `src/{app}/resources/` | 静态资源目录 |
>   | `build/`               | `bundles/`             | 平台打包产物 |
>   | `dist/`                | `dist/`                | 最终发布包  |

### 2.3.1.1 前端资源处理策略

前端编译产物（Vue/React 构建输出）不建议放入 Python 源码目录，推荐以下策略：

**策略一：独立前端目录 + 构建同步（推荐）**

```powershell
# 开发时：前端独立开发
cd frontend
npm run dev

# 构建前端
npm run build  # 输出到 frontend/dist/

# 打包应用时自动同步
pyapp build android  # 自动将 frontend/dist/ 同步到打包目录
```

**开发阶段前后端联调方案**：

由于前端编译产物不在 Python 源码目录，开发阶段需要特殊处理：

**方案 A：前端开发服务器代理（推荐）**

前端开发服务器（Vite/Vue CLI）配置代理，API 请求转发到后端：

```typescript
// frontend/vite.config.ts
export default defineConfig({
  server: {
    proxy: {
      '/api': 'http://localhost:18080',  // API 代理到后端
    }
  }
})
```

```
开发时:
  前端: http://localhost:5173 (Vite dev server)
  后端: http://localhost:18080 (FastAPI)
  
  前端访问 /api/* → 代理到后端 18080 端口
```

**方案 B：开发模式自动同步**

`pyapp dev` 命令启动时自动同步前端资源：

```powershell
pyapp dev android
# → 自动执行: frontend/dist/ → src/myapp/resources/static/
# → 然后启动应用
```

**方案 C：符号链接（Linux/macOS 推荐）**

```bash
# 创建符号链接
ln -s ../../frontend/dist src/myapp/resources/static

# 开发时前端自动更新，后端直接访问
```

**方案 D：代码容错处理**

FastAPI 代码中检查目录是否存在：

```python
# src/myapp/app.py
STATIC_DIR = RESOURCES_DIR / "static"

# 只有目录存在时才挂载
if STATIC_DIR.exists() and any(STATIC_DIR.iterdir()):
    app.mount("/static", StaticFiles(directory=STATIC_DIR), name="static")
else:
    # 开发模式：返回提示信息
    @app.get("/")
    async def index():
        return {
            "message": "Development mode",
            "hint": "Run 'cd frontend && npm run dev' for frontend development",
            "api_docs": "/docs"
        }
```

**策略二：纯后端项目（无前端源码）**

如果前端由其他团队维护，只提供编译产物：

```
myapp/
├── src/myapp/
│   └── resources/
│       └── static/         # 直接放置编译产物（gitignore）
│           ├── index.html
│           └── assets/
└── .gitignore              # 忽略 resources/static/
```

**策略三：CI 中构建前端**

在 CI/CD 中先构建前端，再打包应用：

```yaml
# .github/workflows/build.yml
- name: Build Frontend
  run: |
    cd frontend
    npm install
    npm run build

- name: Build App
  run: pyapp build all  # 自动同步 frontend/dist/
```

#### 2.3.2 `pyapp create`

创建平台特定的项目结构（启动器代码）。

```powershell
pyapp create android        # 生成 Gradle 项目
pyapp create windows        # 生成 Stub 源码
pyapp create linux          # 生成 Shell 脚本
pyapp create all            # 创建所有平台
```

**与** **`build`** **命令的关系**：

| 命令                        | 作用            | 使用场景               |
| ------------------------- | ------------- | ------------------ |
| `pyapp create`            | 仅创建平台项目结构，不构建 | 需要自定义平台配置时         |
| `pyapp build`             | 自动创建缺失结构并构建   | **推荐**：日常开发和 CI/CD |
| `pyapp build --no-create` | 仅构建，不自动创建     | 高级用户，结构已存在时        |

**推荐工作流**：

```powershell
# 方式一：自动模式（推荐）
pyapp build android  # 自动检测并创建缺失的平台项目，然后构建

# 方式二：显式分离（适合需要自定义的场景）
pyapp create android          # 创建项目结构
# 编辑 bundles/android/app/build.gradle.kts 自定义配置
pyapp build android           # 构建
```

**各平台产出**：

| 平台      | 产出目录               | 内容                                             |
| ------- | ------------------ | ---------------------------------------------- |
| Android | `bundles/android/` | Gradle 项目 (settings.gradle.kts, app/, ...)     |
| Windows | `bundles/windows/` | Stub 源码 (app\_stub.c, app\_stub.rc, build.bat) |
| Linux   | `bundles/linux/`   | Shell 脚本 (run.sh, install.sh, app.service)     |

#### 2.3.4 `pyapp run`

安装并运行应用（用于测试）。

```powershell

pyapp run windows           # 运行 exe
pyapp run linux             # systemctl start
```

#### 2.3.5 `pyapp dev`

开发模式，启动文件监听和热重载。

```powershell
pyapp dev android           # 监听 python/ → adb push → 重启应用
pyapp dev windows           # 监听 python/ → 复制 → 重启进程
pyapp dev linux             # 监听 python/ → 复制 → systemctl restart
```

**热重载流程**：

```
文件变化 → 同步代码 → 触发重启 → 新进程加载新代码
    │
    ├─ Android: adb push → am force-stop → am start
    ├─ Windows: 复制文件 → POST /api/restart
    └─ Linux: 复制文件 → systemctl restart
```

#### 2.3.6 `pyapp package`

打包发布版本（签名、优化）。

```powershell
pyapp package android       # 签名 APK
pyapp package windows       # 签名 exe
pyapp package linux         # 打包发布版 tar.gz
```

## 三、配置文件设计

### 3.1 pyproject.toml

使用标准 Python 项目配置文件，遵循 [PEP 621](https://peps.python.org/pep-0621/) 规范。

```toml
[project]
name = "data-collector"
version = "1.2.0"                                    # 应用版本
description = "跨平台数据采集应用"
authors = [{name = "Developer", email = "dev@example.com"}]
requires-python = ">=3.10"
dependencies = [
    "fastapi>=0.115.0",
    "uvicorn>=0.30.0",
    "platformdirs>=4.0.0",
]

[project.optional-dependencies]
dev = [
    "pytest>=7.0.0",
    "httpx>=0.24.0",
]

[build-system]
requires = ["setuptools>=61.0"]
build-backend = "setuptools.build_meta"

[tool.pyapp]
# Python 运行时版本（用于打包）
python_version = "3.12.1"

# 应用配置
app_module = "data_collector"      # Python 模块名（入口）
port = 18080                       # FastAPI 端口

# Android 配置
[tool.pyapp.android]
package_name = "com.example.datacollector"
min_sdk = 24
target_sdk = 34
permissions = [
    "INTERNET",
    "FOREGROUND_SERVICE",
    "RECEIVE_BOOT_COMPLETED",
]
# 可选：自定义 PyPI 仓库（用于 Chaquopy 兼容包）
extra_index_url = "https://jzhenli.github.io/my-pypi/simple"

# Android 特定依赖（覆盖或追加到 project.dependencies）
dependencies = [
    "numpy==1.26.4",  # 指定 Chaquopy 兼容版本
]

# Windows 配置
[tool.pyapp.windows]
deployment = "standalone"          # standalone 或 shared
create_service = true              # 是否创建 Windows 服务
service_name = "DataCollector"     # 服务名称

# Windows 特定依赖
dependencies = [
    "pywin32>=305",    # Windows 服务支持
]

# Linux 配置
[tool.pyapp.linux]
deployment = "shared"              # standalone 或 shared
install_systemd = true             # 是否安装 systemd 服务
service_name = "data-collector"    # 服务名称
```

**配置项说明**：

| 配置项                                  | 来源      | 说明                         |
| ------------------------------------ | ------- | -------------------------- |
| `project.version`                    | PEP 621 | 应用版本号                      |
| `project.dependencies`               | PEP 621 | 运行时依赖（替代 requirements.txt） |
| `tool.pyapp.python_version`          | 自定义     | 打包时使用的 Python 运行时版本        |
| `tool.pyapp.{platform}.dependencies` | 自定义     | 平台特定依赖（追加或覆盖）              |
| `tool.pyapp.android/windows/linux`   | 自定义     | 平台特定配置                     |

**依赖合并策略**：平台特定依赖会覆盖或追加到全局依赖中，相同包名时平台配置优先。

**优势**：

- 遵循 Python 标准，工具兼容性好
- 单文件管理所有配置，减少维护成本
- 支持依赖分组（dev/test/production）
- 可被 pip、setuptools、poetry 等工具识别

### 3.2 静态资源访问

`resources/` 目录用于存放静态资源。前端编译产物建议放在 `frontend/dist/`，构建时自动同步。

```python
# src/myapp/app.py
import os
from pathlib import Path
from fastapi import FastAPI
from fastapi.staticfiles import StaticFiles
from fastapi.responses import FileResponse, JSONResponse

app = FastAPI()

# 获取 resources 目录路径
RESOURCES_DIR = Path(__file__).parent / "resources"

# 静态文件目录（构建时从 frontend/dist/ 同步）
STATIC_DIR = RESOURCES_DIR / "static"

# 开发模式检测
IS_DEV_MODE = os.environ.get("APP_MODE", "production") == "dev"

# 只有目录存在且有内容时才挂载静态文件
if STATIC_DIR.exists() and any(STATIC_DIR.iterdir()):
    app.mount("/static", StaticFiles(directory=STATIC_DIR), name="static")
    FRONTEND_AVAILABLE = True
else:
    FRONTEND_AVAILABLE = False
    if not IS_DEV_MODE:
        print(f"Warning: Frontend not found at {STATIC_DIR}")

@app.get("/")
async def index():
    """返回首页"""
    if FRONTEND_AVAILABLE:
        return FileResponse(STATIC_DIR / "index.html")
    
    # 开发模式提示
    return JSONResponse({
        "message": "Frontend not available",
        "mode": "development" if IS_DEV_MODE else "production",
        "hints": [
            "For development: cd frontend && npm run dev",
            "For production: npm run build && pyapp build <platform>",
            "API documentation: /docs"
        ]
    })

@app.get("/api/health")
async def health():
    """健康检查"""
    return {
        "status": "ok",
        "frontend": "available" if FRONTEND_AVAILABLE else "not_built"
    }

@app.get("/config")
async def get_config():
    """读取配置文件"""
    import yaml
    config_file = RESOURCES_DIR / "config" / "config.yaml"
    if config_file.exists():
        with open(config_file, "r", encoding="utf-8") as f:
            return yaml.safe_load(f)
    return {}
```

**开发与生产环境对比**：

| 环境        | 前端访问方式                   | 后端端口  | 说明                       |
| --------- | ------------------------ | ----- | ------------------------ |
| 开发（前后端分离） | `http://localhost:5173`  | 18080 | Vite 开发服务器，API 代理        |
| 开发（集成调试）  | `http://localhost:18080` | 18080 | 需先 `npm run build` 或符号链接 |
| 生产（打包后）   | `http://localhost:18080` | 18080 | 静态资源已打包到应用内              |

**目录结构对应关系**：

| 源码路径                                     | 打包后路径                              | 访问 URL                        |
| ---------------------------------------- | ---------------------------------- | ----------------------------- |
| `frontend/dist/index.html`               | `resources/static/index.html`      | `GET /`                       |
| `frontend/dist/assets/index.js`          | `resources/static/assets/index.js` | `GET /static/assets/index.js` |
| `src/myapp/resources/config/config.yaml` | `resources/config/config.yaml`     | 通过 API 读取                     |

**构建时同步逻辑**：

```python
# pyapp/core/builder.py
import json
import shutil
from pathlib import Path
from typing import Optional, Dict, Any

def sync_frontend_dist(project_dir: Path, platform: str) -> Optional[Path]:
    """
    将前端编译产物同步到打包目录

    源: frontend/dist/
    目标: bundles/{platform}/app/app/myapp/resources/static/

    Args:
        project_dir: 项目根目录
        platform: 平台名称

    Returns:
        目标目录路径，如果源不存在则返回 None
    """
    from .config import load_config

    frontend_dist = project_dir / "frontend" / "dist"
    config = load_config(project_dir)
    app_module = config.pyapp.app_module

    target_static = (
        project_dir / "bundles" / platform / "app" / "app" /
        app_module / "resources" / "static"
    )

    if not frontend_dist.exists():
        print(f"Warning: frontend/dist/ not found, skipping frontend sync")
        return None

    # 验证前端版本兼容性
    if not validate_frontend_version(frontend_dist, config):
        print(f"Warning: Frontend version mismatch, sync may fail")

    # 同步前端资源
    if target_static.exists():
        shutil.rmtree(target_static)
    shutil.copytree(frontend_dist, target_static)

    print(f"Synced frontend/dist/ → {target_static}")
    return target_static

def validate_frontend_version(frontend_dist: Path, config) -> bool:
    """
    验证前端版本与后端 API 版本的兼容性

    Args:
        frontend_dist: 前端编译产物目录
        config: 应用配置

    Returns:
        是否兼容
    """
    # 检查前端版本信息文件
    version_file = frontend_dist / "version.json"
    if not version_file.exists():
        print("Warning: Frontend version.json not found, skipping version check")
        return True

    try:
        with open(version_file, "r", encoding="utf-8") as f:
            frontend_version = json.load(f)

        # 获取后端版本
        backend_version = config.project.version

        # 比较主版本号
        frontend_major = frontend_version.get("version", "0.0.0").split(".")[0]
        backend_major = backend_version.split(".")[0]

        if frontend_major != backend_major:
            print(f"Warning: Major version mismatch - Frontend: {frontend_major}, Backend: {backend_major}")
            return False

        # 检查 API 版本兼容性
        api_version = frontend_version.get("api_version", "1.0")
        supported_api_versions = ["1.0", "1.1", "2.0"]  # 从配置中读取支持的 API 版本

        if api_version not in supported_api_versions:
            print(f"Warning: API version {api_version} may not be compatible")
            return False

        return True

    except Exception as e:
        print(f"Warning: Failed to validate frontend version: {e}")
        return True  # 验证失败时允许继续，但给出警告

def generate_frontend_version_file(frontend_dir: Path, version: str, api_version: str = "1.0") -> None:
    """
    生成前端版本信息文件

    在前端构建时调用此函数，生成 version.json 文件

    Args:
        frontend_dir: 前端项目目录
        version: 前端版本号
        api_version: API 版本号
    """
    import datetime

    version_info = {
        "version": version,
        "api_version": api_version,
        "build_time": datetime.datetime.now().isoformat(),
        "git_commit": get_git_commit_hash(frontend_dir),
    }

    dist_dir = frontend_dir / "dist"
    dist_dir.mkdir(parents=True, exist_ok=True)

    version_file = dist_dir / "version.json"
    with open(version_file, "w", encoding="utf-8") as f:
        json.dump(version_info, f, indent=2)

    print(f"Generated frontend version file: {version_file}")

def get_git_commit_hash(project_dir: Path) -> Optional[str]:
    """获取 Git 提交哈希"""
    import subprocess

    try:
        result = subprocess.run(
            ["git", "rev-parse", "HEAD"],
            cwd=project_dir,
            capture_output=True,
            text=True,

        return None
```

### 4.2 配置解析模块

```python
# pyapp/core/config.py
import tomllib  # Python 3.11+
import sys
from pathlib import Path
from dataclasses import dataclass
from typing import Optional, List, Dict, Any

# Python < 3.11 兼容
if sys.version_info < (3, 11):
    import tomli as tomllib


@dataclass
class ProjectConfig:
    """项目配置"""
    name: str
    version: str
    description: str = ""
    authors: List[Dict[str, str]] = None
    requires_python: str = ">=3.10"
    dependencies: List[str] = None
    optional_dependencies: Dict[str, List[str]] = None


@dataclass
class PyAppConfig:
    """PyApp 工具配置"""
    python_version: str = "3.12.1"
    app_module: str = ""  # 留空表示自动检测
    port: int = 18080

    # 平台配置
    android: Dict[str, Any] = None
    windows: Dict[str, Any] = None
    linux: Dict[str, Any] = None


@dataclass
class AppConfig:
    """完整应用配置"""
    project: ProjectConfig
    pyapp: PyAppConfig

    @classmethod
    def from_file(cls, path: Path) -> "AppConfig":
        """从 pyproject.toml 加载配置"""
        with open(path, "rb") as f:
            data = tomllib.load(f)

        # 解析 project 段
        project_data = data.get("project", {})
        project = ProjectConfig(
            name=project_data.get("name", ""),
            version=project_data.get("version", "0.1.0"),
            description=project_data.get("description", ""),
            authors=project_data.get("authors", []),
            requires_python=project_data.get("requires-python", ">=3.10"),
            dependencies=project_data.get("dependencies", []),
            optional_dependencies=project_data.get("optional-dependencies", {}),
        )

        # 解析 tool.pyapp 段
        pyapp_data = data.get("tool", {}).get("pyapp", {})

        # 自动检测 app_module（如果未配置）
        app_module = pyapp_data.get("app_module", "")
        if not app_module:
            app_module = cls._detect_app_module(path.parent, project.name)

        pyapp = PyAppConfig(
            python_version=pyapp_data.get("python_version", "3.12.1"),
            app_module=app_module,
            port=pyapp_data.get("port", 18080),
            android=pyapp_data.get("android", {}),
            windows=pyapp_data.get("windows", {}),
            linux=pyapp_data.get("linux", {}),
        )

        config = cls(project=project, pyapp=pyapp)
        config._validate(path.parent)
        return config

    @staticmethod
    def _detect_app_module(project_dir: Path, project_name: str) -> str:
        """自动检测 Python 模块名"""
        src_dir = project_dir / "src"
        if src_dir.exists():
            packages = [d for d in src_dir.iterdir()
                       if d.is_dir() and (d / "__init__.py").exists()]
            if len(packages) == 1:
                return packages[0].name
        return project_name.replace("-", "_")

    def _validate(self, project_dir: Path):
        """验证配置有效性"""
        # 1. 验证模块路径存在
        module_path = project_dir / "src" / self.pyapp.app_module
        if not module_path.exists():
            raise ValueError(f"Module '{self.pyapp.app_module}' not found")
        if not (module_path / "__main__.py").exists():
            raise ValueError(f"__main__.py not found in {self.pyapp.app_module}")

        # 2. 验证 Python 版本有效性
        self._validate_python_version()

        # 3. 验证平台特定配置
        self._validate_platform_configs()

        # 4. 验证依赖配置
        self._validate_dependencies()

    def _validate_python_version(self):
        """验证 Python 版本有效性"""
        import re
        from packaging import version

        python_version = self.pyapp.python_version

        # 验证版本格式
        pattern = r"^\d+\.\d+\.\d+$"
        if not re.match(pattern, python_version):
            raise ValueError(
                f"Invalid Python version format: {python_version}. "
                "Expected format: X.Y.Z (e.g., 3.12.1)"
            )

        # 验证版本范围
        min_version = version.parse("3.8.0")
        max_version = version.parse("3.13.0")
        current_version = version.parse(python_version)

        if current_version < min_version:
            raise ValueError(
                f"Python version {python_version} is too old. "
                f"Minimum supported version is {min_version}"
            )

        if current_version >= max_version:
            print(f"Warning: Python version {python_version} may not be fully tested")

        # 验证 Chaquopy 支持的版本（Android 平台）
        if self.pyapp.android:
            chaquopy_versions = ["3.8", "3.9", "3.10", "3.11", "3.12"]
            major_minor = f"{current_version.major}.{current_version.minor}"
            if major_minor not in chaquopy_versions:
                raise ValueError(
                    f"Python {major_minor} is not supported by Chaquopy for Android. "
                    f"Supported versions: {', '.join(chaquopy_versions)}"
                )

    def _validate_platform_configs(self):
        """验证平台特定配置"""
        # Android 平台配置验证
        if self.pyapp.android:
            package_name = self.pyapp.android.get("package_name", "")
            if not package_name:
                raise ValueError(
                    "Android package_name is required. "
                    "Add 'package_name' to [tool.pyapp.android] in pyproject.toml"
                )

            # 验证包名格式
            import re
            if not re.match(r"^[a-z][a-z0-9_]*(\.[a-z][a-z0-9_]*)+$", package_name):
                raise ValueError(
                    f"Invalid Android package name: {package_name}. "
                    "Expected format: com.example.myapp"
                )

            # 验证 SDK 版本
            min_sdk = self.pyapp.android.get("min_sdk", 24)
            target_sdk = self.pyapp.android.get("target_sdk", 34)
            if min_sdk > target_sdk:
                raise ValueError(
                    f"min_sdk ({min_sdk}) cannot be greater than target_sdk ({target_sdk})"
                )

        # Windows 平台配置验证
        if self.pyapp.windows:
            deployment = self.pyapp.windows.get("deployment", "standalone")
            if deployment not in ["standalone", "shared"]:
                raise ValueError(
                    f"Invalid Windows deployment mode: {deployment}. "
                    "Expected: 'standalone' or 'shared'"
                )

        # Linux 平台配置验证
        if self.pyapp.linux:
            deployment = self.pyapp.linux.get("deployment", "shared")
            if deployment not in ["standalone", "shared"]:
                raise ValueError(
                    f"Invalid Linux deployment mode: {deployment}. "
                    "Expected: 'standalone' or 'shared'"
                )

    def _validate_dependencies(self):
        """验证依赖配置"""
        import re
        from packaging import requirements

        # 验证全局依赖格式
        for dep in self.project.dependencies or []:
            try:
                # 验证依赖规范格式
                req = requirements.Requirement(dep)
            except Exception as e:
                raise ValueError(f"Invalid dependency format '{dep}': {e}")

        # 验证平台特定依赖
        for platform in ["android", "windows", "linux"]:
            platform_config = getattr(self.pyapp, platform, {})
            platform_deps = platform_config.get("dependencies", [])
            for dep in platform_deps:
                try:
                    req = requirements.Requirement(dep)
                except Exception as e:
                    raise ValueError(
                        f"Invalid {platform} dependency format '{dep}': {e}"
                    )

    def to_dict(self) -> Dict[str, Any]:
        """转换为字典（供模板渲染使用）"""
        return {
            "project": {
                "name": self.project.name,
                "version": self.project.version,
                "description": self.project.description,
                "dependencies": self.project.dependencies,
            },
            "tool": {
                "pyapp": {
                    "python_version": self.pyapp.python_version,
                    "app_module": self.pyapp.app_module,
                    "port": self.pyapp.port,
                    "android": self.pyapp.android,
                    "windows": self.pyapp.windows,
                    "linux": self.pyapp.linux,
                }
            }
        }


def load_config(project_dir: Path) -> AppConfig:
    """加载项目配置"""
    config_file = project_dir / "pyproject.toml"
    if not config_file.exists():
        raise FileNotFoundError(f"pyproject.toml not found in {project_dir}")
    return AppConfig.from_file(config_file)
```

```python
import logging

class PyAppLogger:
    """PyApp 日志管理器"""

    def __init__(self, name: str = "pyapp", log_dir: Optional[Path] = None):
        """
        初始化日志管理器

        Args:
            name: 日志器名称
            log_dir: 日志文件目录，默认为 ~/.pyapp/logs/
        """
        self.name = name
        self.log_dir = log_dir or Path.home() / ".pyapp" / "logs"
        self.log_dir.mkdir(parents=True, exist_ok=True)

        # 创建日志器
        self.logger = logging.getLogger(name)
        self.logger.setLevel(logging.DEBUG)

        # 避免重复添加 handler
        if not self.logger.handlers:
            self._setup_handlers()

    def _setup_handlers(self):
        """设置日志处理器"""
        # 控制台处理器
        console_handler = logging.StreamHandler(sys.stdout)
        console_handler.setLevel(logging.INFO)
        console_format = logging.Formatter(
            '%(levelname)s: %(message)s',
            datefmt='%H:%M:%S'
        )
        console_handler.setFormatter(console_format)
        self.logger.addHandler(console_handler)

        # 文件处理器
        log_file = self.log_dir / f"{self.name}_{datetime.now().strftime('%Y%m%d')}.log"
        file_handler = logging.FileHandler(log_file, encoding='utf-8')
        file_handler.setLevel(logging.DEBUG)
        file_format = logging.Formatter(
            '%(asctime)s - %(name)s - %(levelname)s - %(message)s',
            datefmt='%Y-%m-%d %H:%M:%S'
        )
        file_handler.setFormatter(file_format)
        self.logger.addHandler(file_handler)

    def set_level(self, level: str):
        """
        设置日志级别

        Args:
            level: 日志级别 (DEBUG/INFO/WARNING/ERROR)
        """
        level_map = {
            "DEBUG": logging.DEBUG,
            "INFO": logging.INFO,
            "WARNING": logging.WARNING,
            "ERROR": logging.ERROR,
        }
        self.logger.setLevel(level_map.get(level.upper(), logging.INFO))

    def debug(self, message: str):
        """记录调试信息"""
        self.logger.debug(message)

    def info(self, message: str):
        """记录一般信息"""
        self.logger.info(message)

    def warning(self, message: str):
        """记录警告信息"""
        self.logger.warning(message)

    def error(self, message: str, exc_info: bool = False):
        """
        记录错误信息

        Args:
            message: 错误消息
            exc_info: 是否包含异常堆栈信息
        """
        self.logger.error(message, exc_info=exc_info)

    def success(self, message: str):
        """记录成功信息（带颜色标记）"""
        # 使用 INFO 级别，但在控制台输出时添加 ✓ 前缀

            total_steps: 总步骤数
```

### 4.2.2 运行时管理模块

```python
# pyapp/core/runtime.py
import hashlib
import shutil
import tarfile
import zipfile
from pathlib import Path
from dataclasses import dataclass
from typing import Optional
from urllib.request import urlopen, Request
from urllib.error import URLError

from .cache import CacheManager
from .errors import DownloadError, VerificationError


@dataclass
class RuntimeSource:
    """运行时下载源"""
    url_template: str
    strip_components: int = 1


# 各平台 Python 运行时下载源
RUNTIME_SOURCES = {
    "windows": RuntimeSource(
        url_template="https://www.python.org/ftp/python/{version}/python-{version}-embed-amd64.zip",
    ),
    "linux": RuntimeSource(
        url_template=(
            "https://github.com/indygreg/python-build-standalone/releases/download/"
            "{version}/cpython-{version}+{build}-x86_64-unknown-linux-gnu-install_only.tar.gz"
        ),
        strip_components=1,
    ),
}

# 已知的 PBS 版本映射
PBS_VERSIONS = {
    "3.12.1": "20240107",
    "3.11.7": "20240107",
    "3.10.13": "20240107",
}


class RuntimeManager:
    """Python 运行时管理器"""

    def __init__(self, cache_dir: Optional[Path] = None):
        self.cache = CacheManager(cache_dir)

    def get_runtime(self, platform: str, version: str, target_dir: Path) -> Path:
        """获取 Python 运行时"""
        cache_key = f"runtime-{platform}-{version}"
        cached_path = self.cache.get(cache_key)

        if cached_path:
            print(f"Using cached runtime: {cached_path}")
            return self._extract_runtime(cached_path, target_dir, platform)

        print(f"Downloading Python {version} for {platform}...")
        downloaded_file = self._download_runtime(platform, version)

        if not self._verify_runtime(platform, downloaded_file):
            raise VerificationError(f"Runtime verification failed")

        self.cache.put(cache_key, downloaded_file)
        return self._extract_runtime(downloaded_file, target_dir, platform)

    def _download_runtime(self, platform: str, version: str) -> Path:
        """下载运行时文件"""
        source = RUNTIME_SOURCES.get(platform)
        if not source:
            raise ValueError(f"Unsupported platform: {platform}")

        if platform == "linux":
            build = PBS_VERSIONS.get(version, "20240107")
            url = source.url_template.format(version=version, build=build)
        else:
            url = source.url_template.format(version=version)

        try:
            request = Request(url, headers={"User-Agent": "PyApp-CLI/1.0"})
            with urlopen(request, timeout=300) as response:
                total_size = int(response.headers.get("Content-Length", 0))
                downloaded = 0

                temp_file = self.cache.temp_dir / f"python-{platform}-{version}.tmp"
                with open(temp_file, "wb") as f:
                    while True:
                        chunk = response.read(8192)
                        if not chunk:
                            break
                        f.write(chunk)
                        downloaded += len(chunk)
                        if total_size:
                            print(f"\rDownloading: {(downloaded/total_size)*100:.1f}%", end="")

                print()
                return temp_file

        except URLError as e:
            raise DownloadError(f"Failed to download runtime: {e}")

    def _verify_runtime(self, platform: str, file_path: Path) -> bool:
        """验证运行时文件完整性"""
        try:
            if platform == "windows":
                with zipfile.ZipFile(file_path, "r") as zf:
                    return "python.exe" in zf.namelist()
            else:
                with tarfile.open(file_path, "r:gz") as tf:
                    return any("bin/python" in n for n in tf.getnames())
        except Exception:
            return False

    def _extract_runtime(self, archive_path: Path, target_dir: Path, platform: str) -> Path:
        """解压运行时"""
        target_dir.mkdir(parents=True, exist_ok=True)

        if platform == "windows":
            with zipfile.ZipFile(archive_path, "r") as zf:
                zf.extractall(target_dir)
        else:
            source = RUNTIME_SOURCES[platform]
            with tarfile.open(archive_path, "r:gz") as tf:
                for member in tf.getmembers():
                    parts = member.name.split("/")
                    if len(parts) > source.strip_components:
                        member.name = "/".join(parts[source.strip_components:])
                        tf.extract(member, target_dir)

        return target_dir
```

**运行时下载源**：

| 平台      | 来源                      | 大小     |
| ------- | ----------------------- | ------ |

### 4.2.2 缓存管理模块

```python
# pyapp/core/cache.py
import json
import shutil
from pathlib import Path
from datetime import datetime, timedelta
from typing import Optional, Dict, Any


class CacheManager:
    """缓存管理器"""

    DEFAULT_CACHE_DIR = Path.home() / ".pyapp" / "cache"
    CACHE_EXPIRE_DAYS = 30

    def __init__(self, cache_dir: Optional[Path] = None):
        self.cache_dir = cache_dir or self.DEFAULT_CACHE_DIR
        self.runtimes_dir = self.cache_dir / "runtimes"
        self.packages_dir = self.cache_dir / "packages"
        self.temp_dir = self.cache_dir / "temp"
        self.metadata_file = self.cache_dir / "metadata.json"

        self.runtimes_dir.mkdir(parents=True, exist_ok=True)
        self.packages_dir.mkdir(parents=True, exist_ok=True)
        self.temp_dir.mkdir(parents=True, exist_ok=True)

        self.metadata = self._load_metadata()

    def _load_metadata(self) -> Dict[str, Any]:
        if self.metadata_file.exists():
            with open(self.metadata_file, "r") as f:
                return json.load(f)
        return {"entries": {}}

    def _save_metadata(self):
        with open(self.metadata_file, "w") as f:
            json.dump(self.metadata, f, indent=2)

    def get(self, key: str) -> Optional[Path]:
        """获取缓存项"""
        entry = self.metadata["entries"].get(key)
        if not entry:
            return None

        cached_time = datetime.fromisoformat(entry["timestamp"])
        if datetime.now() - cached_time > timedelta(days=self.CACHE_EXPIRE_DAYS):
            self.delete(key)
            return None

        cached_path = Path(entry["path"])
        return cached_path if cached_path.exists() else None

    def put(self, key: str, source_path: Path) -> Path:
        """添加缓存项"""
        target_dir = self.runtimes_dir if "runtime" in key else self.packages_dir
        target_path = target_dir / source_path.name

        shutil.move(str(source_path), str(target_path))

        self.metadata["entries"][key] = {
            "path": str(target_path),
            "timestamp": datetime.now().isoformat(),
            "size": target_path.stat().st_size,
        }
        self._save_metadata()

        return target_path

    def delete(self, key: str) -> bool:
        """删除缓存项"""
        entry = self.metadata["entries"].get(key)
        if not entry:
            return False

        cached_path = Path(entry["path"])
        if cached_path.exists():
            cached_path.unlink()

        del self.metadata["entries"][key]
        self._save_metadata()
        return True

    def clear_all(self):
        """清空所有缓存"""
        for key in list(self.metadata["entries"].keys()):
            self.delete(key)

    def get_cache_size(self) -> int:
        """获取缓存总大小"""
        return sum(e.get("size", 0) for e in self.metadata["entries"].values())
```

```python
# pyapp/platforms/base.py
from abc import ABC, abstractmethod
from pathlib import Path
from dataclasses import dataclass
from typing import Optional, List

@dataclass
class BuildResult:
    """构建结果"""
    success: bool
    output_path: Optional[Path] = None
    error_message: str = ""

class BasePlatform(ABC):
    """平台基类"""
    
    name: str = ""
    description: str = ""
    
    @abstractmethod
    def check_environment(self) -> tuple[bool, List[str]]:
        """
        检查环境是否满足要求
        
        Returns:
            (是否满足, 缺失项列表)
        """
        pass
    
    @abstractmethod
    def create(self, project_dir: Path, config: dict) -> None:
        """
        创建平台项目结构
        
        Args:
            project_dir: 项目根目录
            config: 解析后的配置
        """
        pass
    
    @abstractmethod
    def build(self, project_dir: Path, config: dict, build_type: str = "debug") -> BuildResult:
        """
        构建平台包
        
        Args:
            project_dir: 项目根目录
            config: 解析后的配置
            build_type: debug 或 release
            
        Returns:
            构建结果
        """
        pass
    
    @abstractmethod
    def run(self, project_dir: Path, config: dict) -> None:
        """
        运行应用
        """
        pass
    
    @abstractmethod
    def dev(self, project_dir: Path, config: dict) -> None:
        """
        开发模式（文件监听 + 热重载）
        """
        pass
    
    @abstractmethod
    def package(self, project_dir: Path, config: dict) -> BuildResult:
        """
        打包发布版
        """
        pass
    
    # ===== 公共方法 =====
    
    def install_dependencies(self, project_dir: Path, config: dict, platform: str) -> Path:
        """
        安装 pip 依赖到 bundles/{platform}/app/app_packages/

        Args:
            project_dir: 项目根目录
            config: 解析后的配置
            platform: 平台名称 (android/windows/linux)

        Returns:
            app_packages 目录路径
        """
        import subprocess
        import sys

        # 从 pyproject.toml 读取依赖
        dependencies = config.get("project", {}).get("dependencies", [])
        platform_config = config.get("tool", {}).get("pyapp", {}).get(platform, {})
        platform_dependencies = platform_config.get("dependencies", [])

        # 合并依赖（平台特定依赖优先）
        all_dependencies = dependencies + platform_dependencies

        target = project_dir / "bundles" / platform / "app" / "app_packages"
        target.mkdir(parents=True, exist_ok=True)

        if not all_dependencies:
            return target

        # 根据平台选择不同的安装策略
        if platform == "android":
            # Android 平台使用 Chaquopy 兼容的包
            self._install_android_dependencies(all_dependencies, target, platform_config)
        else:
            # Windows/Linux 平台
            self._install_native_dependencies(all_dependencies, target, platform)

        return target

    def _install_android_dependencies(self, dependencies: list, target: Path, platform_config: dict) -> None:
        """
        安装 Android 平台依赖（考虑 Chaquopy 兼容性）

        Args:
            dependencies: 依赖列表
            target: 目标目录
            platform_config: 平台配置
        """
        import subprocess

        # Chaquopy 支持的 Python 版本
        chaquopy_python_versions = ["3.8", "3.9", "3.10", "3.11", "3.12"]

        # 使用 Chaquopy 的 PyPI 仓库
        extra_index_url = platform_config.get("extra_index_url",
            "https://chaquo.com/chaquopy/maven/org/python/pypi/simple")

        # 安装依赖
        cmd = [
            "pip", "install"
        ] + dependencies + [
            "--target", str(target),
            "--extra-index-url", extra_index_url,
            "--no-deps",  # 避免安装不兼容的依赖
        ]

        result = subprocess.run(cmd, capture_output=True, text=True)
        if result.returncode != 0:
            # 如果安装失败，尝试逐个安装
            for dep in dependencies:
                subprocess.run([
                    "pip", "install", dep,
                    "--target", str(target),
                    "--extra-index-url", extra_index_url,
                ], check=False)

    def _install_native_dependencies(self, dependencies: list, target: Path, platform: str) -> None:
        """
        安装 Windows/Linux 平台依赖

        Args:
            dependencies: 依赖列表
            target: 目标目录
            platform: 平台名称
        """
        import subprocess
        import sys

        # 检测当前平台
        current_platform = sys.platform
        is_cross_compile = (
            (platform == "windows" and current_platform != "win32") or
            (platform == "linux" and current_platform != "linux")
        )

        if is_cross_compile:
            # 跨平台编译：需要指定目标平台
            print(f"Warning: Cross-platform compilation detected (building {platform} on {current_platform})")
            print("Note: Some packages with C extensions may not work correctly")

            # 尝试使用纯 Python 实现或预编译的 wheel
            cmd = [
                "pip", "install"
            ] + dependencies + [
                "--target", str(target),
                "--only-binary=:all:",  # 只使用预编译的 wheel
            ]
        else:
            # 本地编译：正常安装
            cmd = [
                "pip", "install"
            ] + dependencies + [
                "--target", str(target),
            ]

        subprocess.run(cmd, check=True)
    
    def sync_source_code(self, project_dir: Path, platform: str) -> Path:
        """
        同步 Python 源码到 bundles/{platform}/app/app/
        
        包括:
        - Python 代码 (src/myapp/*.py)
        - 配置文件 (src/myapp/resources/config/)
        
        注意: 前端编译产物 (frontend/dist/) 由 sync_frontend_dist() 单独处理
        
        Args:
            project_dir: 项目根目录
            platform: 平台名称
            
        Returns:
            目标目录路径
        """
        import shutil
        
        src_dir = project_dir / "src"
        target_dir = project_dir / "bundles" / platform / "app" / "app"
        
        if target_dir.exists():
            shutil.rmtree(target_dir)
        
        shutil.copytree(src_dir, target_dir)
        return target_dir
    
    def sync_frontend_dist(self, project_dir: Path, platform: str) -> Path:
        """
        同步前端编译产物到打包目录
        
        源: frontend/dist/
        目标: bundles/{platform}/app/app/myapp/resources/static/
        
        Args:
            project_dir: 项目根目录
            platform: 平台名称
            
        Returns:
            目标目录路径，如果源不存在则返回 None
        """
        import shutil
        
        frontend_dist = project_dir / "frontend" / "dist"
        config = load_config(project_dir)
        app_module = config.pyapp.app_module
        
        target_static = (
            project_dir / "bundles" / platform / "app" / "app" / 
            app_module / "resources" / "static"
        )
        
        if frontend_dist.exists():
            if target_static.exists():
                shutil.rmtree(target_static)
            shutil.copytree(frontend_dist, target_static)
            return target_static
        return None
    
    def get_resources_dir(self, project_dir: Path) -> Path:
        """
        获取静态资源目录路径
        
        Returns:
            resources 目录路径 (src/myapp/resources/)
        """
        config = load_config(project_dir)
        app_module = config.pyapp.app_module
        return project_dir / "src" / app_module / "resources"
    
    def get_python_version(self, config: dict) -> str:
        """获取 Python 版本"""
        return config.get("tool", {}).get("pyapp", {}).get("python_version", "3.12.1")
    
    def get_app_version(self, config: dict) -> str:
        """获取应用版本"""
        return config.get("project", {}).get("version", "0.1.0")
```

```python
def setup_android():
    """安装 Android 开发环境"""
    
    # 1. 安装 JDK 17
    jdk_dir = Path.home() / ".android-jdk"
    if not jdk_dir.exists():
        print("Installing JDK 17...")
        download_jdk(jdk_dir)
    
    # 2. 安装 Android SDK
    sdk_dir = Path.home() / ".android-sdk"
    if not sdk_dir.exists():
        print("Installing Android SDK...")
        download_android_sdk(sdk_dir)
        install_sdk_packages(sdk_dir)
    
    # 3. 设置环境变量
    print("\nEnvironment variables:")
    print(f'  $env:JAVA_HOME = "{jdk_dir}"')
    print(f'  $env:ANDROID_HOME = "{sdk_dir}"')
```

## 十二、总结

本方案旨在将分散的构建脚本整合为统一的 CLI 工具，提供类似 Briefcase 的跨平台开发体验：

**核心价值**：

1. **统一命令** - 一套命令操作三个平台
2. **统一配置** - `pyproject.toml` 集中管理
3. **自动化** - 环境安装、构建、部署全自动
4. **开发友好** - 热重载、增量更新、一键调试

**与 Briefcase 的差异化**：

- 面向服务型应用（FastAPI + WebView）
- 支持热更新（VERSION 文件机制）
- 更灵活的部署模式（standalone/shared）
